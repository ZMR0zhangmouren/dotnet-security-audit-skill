.NET Security Audit Pipeline Architecture

From route mapping to exploit-chain aggregation and final quality report

流程主线

  • • 先建模入口与鉴权，再做漏洞初筛和路由追踪闭合。
  • • framework 专项提供漏洞专项可复用的前置事实。
  • • 最终通过 exploit-chain 与 pipeline 输出统一报告。

证据约束

  • • 高风险结论必须回链 route_id、trace_status、sink evidence。
  • • 证据不足时仅输出待验证或环境依赖，不强行下结论。
  • • 汇总阶段保留冲突证据，避免静默覆盖。

产物矩阵

  • • O1~O8 覆盖 route/auth/trace/framework/vuln/poc/cross/report。
  • • 每阶段建议落盘 index 时间戳文档，保证可追溯。
  • • 统一执行状态可降低“未发现即未记录”的盲区。
