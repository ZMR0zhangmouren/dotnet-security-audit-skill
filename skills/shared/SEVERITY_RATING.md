# 漏洞统一分级标准（.NET）

## 严重等级计算公式

```text
Score = R × 0.40 + I × 0.35 + C × 0.25
CVSS 映射 = Score / 3.0 × 10.0
```

## 三维评分标准

### 可达性 (R)

- 3：无需认证，HTTP/Hub/GraphQL 直接可达
- 2：需要普通用户认证
- 1：需要管理员、内网或特定角色
- 0：不可达或 trace_status=UNRESOLVED

### 影响范围 (I)

- 3：RCE、任意文件写入、完全数据泄露、租户突破
- 2：敏感数据泄露、越权、部分任意读写、高价值业务影响
- 1：有限信息泄露、局部状态异常、低影响配置暴露
- 0：无实际安全影响

### 利用复杂度 (C)

- 3：低复杂度，单请求或低成本触发
- 2：中复杂度，需构造 payload、特定角色或多步操作
- 1：高复杂度，需要竞态、链式利用或严格环境条件
- 0：不可利用

| ---- | ---- | ---- |
| C | 2.70 - 3.00 | 9.0 - 10.0 |

| M | 1.20 - 2.09 | 4.0 - 6.9 |
| L | 0.10 - 1.19 | 0.1 - 3.9 |

## 漏洞编号规范

格式：{等级前缀}-{类型代码}-{序号}

类型代码建议：

- SQL / NOSQL / CMD / SSRF / XSS / FILE / UPLOAD / WRITE / ARCHIVE
- XXE / DESER / TPL / LDAP / EXPR
- AUTH / CSRF / SESS / CFG / CRYPTO / LOGIC / FS / LOG

## 可利用性标注

| 标记 | 含义 |
| ---- | ---- |
| ✅ 已确认可利用 | 证据链闭合，执行点、可控性与前置条件均已成立 |
| ⚠️ 待验证 | 存在风险，但 trace 或关键证据不足 |
| ❌ 不可利用 | 有效防护成立或关键执行点无法到达 |
| 🔍 环境依赖 | 需依赖宿主环境、部署配置或外部组件条件 |
